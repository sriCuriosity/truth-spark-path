## NEW SUPABASE EDGE FUNCTIONS

Create these Edge Functions. Each validates the request using either the user's JWT or their API token from `api_tokens`.

### `POST /functions/v1/sdk/activity`

Accepts batched activity from the browser extension or desktop SDK.

```typescript
// Expected request body:
{
  activities: [
    {
      platform: 'github' | 'youtube' | 'linkedin' | 'medium' | 'browser',
      activity_type: string,
      metadata: Record<string, any>,
      source_url: string,
      occurred_at: string
    }
  ]
}

// Logic:
// 1. Validate auth (JWT or Bearer token from api_tokens)
// 2. Insert each activity into external_activity_log
// 3. For each activity, run classification:
//    - If platform === 'github' and activity_type === 'push': create cortex_suggestions row
//    - If platform === 'youtube' and watch_percentage > 70: create cortex_suggestions row
//    - If platform === 'linkedin' and activity_type === 'article_published': create cortex_suggestions row
//    - Otherwise: log only, no suggestion
// 4. Return { received: N, suggestions_created: M }
```

### `POST /functions/v1/sdk/token`

Creates a personal API token for SDK authentication.

```typescript
// Auth: JWT required
// Logic:
// 1. Generate a secure random token (crypto.randomUUID() + timestamp)
// 2. Hash it with SHA-256, store hash in api_tokens
// 3. Return the raw token ONCE — never stored in plain text
// Response: { token: 'nexus_...', label, expires_at }
```

### `POST /functions/v1/sdk/classify`

Classifies a page/activity and returns suggested Cortex entry type and domains.

```typescript
// Auth: JWT or Bearer token
// Body: { url: string, title: string, content_snippet: string, platform: string }
// Logic: Call Anthropic API (claude-haiku) with prompt:
//   "Classify this web page activity for a learning portfolio system.
//    Return JSON: { is_educational: boolean, domains: string[], 
//    suggested_entry_type: 'action'|'experiment'|'contribution'|'perspective_shift'|null,
//    relevance_text: string (max 20 words) }"
// Response: parsed JSON from AI
```

### `GET /functions/v1/sdk/suggestions/pending`

Returns unactioned suggestions for the authenticated user.

```typescript
// Auth: JWT or Bearer token
// Query params: limit (default 10), platform (optional filter)
// Returns: cortex_suggestions rows where status = 'pending', ordered by created_at desc
```

### `POST /functions/v1/sdk/suggestions/:id/accept`

Converts a suggestion into a real Cortex entry.

```typescript
// Auth: JWT
// Body: { entry_type: string, title: string, body: string, domains: string[] }
// Logic:
// 1. Verify suggestion belongs to user
// 2. Insert into cortex_entries using body fields (override suggestion defaults if provided)
// 3. Update suggestion status to 'accepted', set actioned_at
// 4. Return created cortex_entries row
```

### `POST /functions/v1/sdk/suggestions/:id/discard`

```typescript
// Auth: JWT
// Logic: Update suggestion status to 'discarded', set actioned_at
// Return: { success: true }
```

## EXTENSION COORDINATOR SERVICE (client-side)

Create `src/services/extensionCoordinator.ts`:

```typescript
// Handles communication between the NEXUS web app and the browser extension
// Uses window.postMessage protocol (same origin) when popup is embedded
// Uses BroadcastChannel for cross-tab coordination

export const ExtensionCoordinator = {
  // Send current auth token to extension popup when it loads
  sendAuthToExtension(token: string) {
    window.postMessage({ type: 'NEXUS_AUTH_TOKEN', token }, '*')
  },

  // Listen for activity submitted from extension popup
  onActivitySubmitted(callback: (data: any) => void) {
    window.addEventListener('message', (event) => {
      if (event.data?.type === 'NEXUS_ACTIVITY_SUBMITTED') {
        callback(event.data.payload)
      }
    })
  },

  // Notify extension when Cortex entry is created (so popup can update)
  notifyCortexUpdated(entry: any) {
    const channel = new BroadcastChannel('nexus_cortex')
    channel.postMessage({ type: 'CORTEX_ENTRY_CREATED', entry })
    channel.close()
  }
}
```

Import and initialise `ExtensionCoordinator` in the root layout component. Call `sendAuthToExtension` after successful login.
